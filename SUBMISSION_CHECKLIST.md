# Submission Checklist

Use this checklist before turning the project in so the repo lines up with the
grading rubric.

## Code And Repo Items

- [x] Project runs with beginner-friendly tools.
- [x] Piano notes can be played from the interface.
- [x] Recording, playback, save, and load features are implemented.
- [x] GitHub repository link is included in project code and documentation.
- [x] README explains the project and how to run it.
- [x] Design document is included in the repository.
- [x] Code comments and module headers were expanded.

## Team Items To Finish Manually

- [ ] Replace placeholder team-member names in `README.md` and code headers.
- [ ] Add exact teammate ownership notes for files/functions/classes.
- [ ] Make sure each team member has at least one used commit in GitHub.
- [ ] Confirm Trello has weekly updates in the required format.
- [ ] Confirm the final design document matches what the team wants to present.
- [ ] Practice the code-check demo and decide who explains each feature.
- [ ] Submit the final project correctly on Canvas.

## Demo Talking Points

- Explain how note playback works.
- Show how the app records note timing.
- Show saving and loading with `pickle`.
- Point out the Pillow-generated banner as a multimedia element.
- Mention that missing WAV files can be regenerated automatically.
